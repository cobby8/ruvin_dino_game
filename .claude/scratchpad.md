#### A. 게임 밸런스 분석

**1) 점프 높이 vs 장애물/적 크기 비율**

| 항목 | 수치 | 화면상 크기 |
|------|------|-----------|
| LOW_VELOCITY (1단 점프) | -495 | 최대높이 ~153px |
| DOUBLE_VELOCITY (2단 점프) | -550 | 추가 ~190px (총 ~315px) |
| 공룡 스케일 | 0.28 (x1.5) | ~144px 높이 |
| 장애물 obstacleScale | 난이도별 0.8~1.2 | 장애물 비례 |
| 적 크기 | 스케일 0.12 (고정) | ~86px |

- 판정: 1단 점프(153px)로 일반 장애물/적 넘기 충분. 비행 적(flyHeight 0.45~0.5)은 2단 점프 필요.
- 문제 없음. 6살 기준 적절한 난이도.

**2) 난이도 간 차이 체감도**

| 난이도 | 속도 | 간격(min~max) | 하트 | 장애물크기 | 2단점프 |
|--------|------|-------------|------|----------|--------|
| 아기(1) | 130~260 | 2500~4000 | 5개 | 0.8x | 무제한 |
| 꼬마(2) | 156~325 | 2000~3500 | 4개 | 0.9x | 무제한 |
| 씩씩(3) | 182~390 | 1800~3000 | 3개 | 1.0x | 무제한 |
| 용감(4) | 208~455 | 1500~2500 | 2개 | 1.1x | 3회 제한 |
| 전설(5) | 234~520 | 1200~2000 | 1개 | 1.2x | 불가 |

- 판정: 속도 차이 약 20% 씩 증가 - 적절함
- 문제: 아기->꼬마 간 차이가 거의 안 느껴질 수 있음 (하트 5->4는 체감 낮음)
- 문제: 전설(5)은 하트 1개+2단점프 불가로 6살에겐 거의 불가능 - 의도적 설계로 판단

**3) 게임 속도 증가율**

- SPEED_INCREMENT = 5 (10점마다 +5px/s)
- 아기 난이도 기준: 130에서 시작, 10점마다 +5 -> 100점 넘으면 180 (50점 증가)
- 판정: 점진적이고 적절. 후반 속도감 체감 가능.

**4) 하트 수 + 무적 시간**

| 난이도 | 하트 | 무적시간 |
|--------|------|---------|
| 아기 | 5개 | 3초 |
| 꼬마 | 4개 | 2.5초 |
| 씩씩 | 3개 | 2초 |
| 용감 | 2개 | 1.5초 |
| 전설 | 1개 | 1초 |

- 판정: 아기공룡 5하트+3초 무적은 6살에게 매우 관대. 좋음.
- 잠재 문제: 하트 아이템 출현율 5%로 낮아서 회복 기회가 적음.

**5) 스테이지 클리어 조건 (target 장애물 수)**

| 월드 | 기본 target | 아기(x0.5) | 씩씩(x1.0) | 전설(x1.5) |
|------|------------|-----------|-----------|-----------|
| W1 첫걸음 | 50 | 25 | 50 | 75 |
| W1 왕 | 90 | 45 | 90 | 135 |
| W3 수호자 | 120 | 60 | 120 | 180 |
| W6 끝 | 150 | 75 | 150 | 225 |

- 문제: target이 "넘은 장애물 수"인데 50~150은 상당히 높음.
- 아기공룡 25개도 2~3분 걸림. 6살에겐 집중력 한계.
- 개선 제안: W1은 10~20, 후반도 최대 50~80 정도로 하향 필요 (P1)

**6) 콤보/점수 시스템**

- 2초 내 연속 장애물 넘기 = 콤보 (최대 5배)
- 문제: 장애물 간격 최소 1200~2500ms인데 콤보 타임아웃이 2000ms
- 높은 난이도에서만 콤보 유지 가능, 낮은 난이도에선 거의 불가능
- 개선 제안: 콤보 타임아웃을 난이도별 장애물 최대 간격과 맞추기 (P2)

**7) 아이템 출현 확률**

| 아이템 | 확률 | 비고 |
|--------|------|------|
| 별 라인 | 40% | 적절 |
| 하트 | 5% | 너무 낮음 - 아기 난이도에서도 회복 어려움 |
| 물음표블록 | 15% | 적절 |
| 스프링 | 3~10% (스테이지별) | 적절 |
| 부스트 | 0~8% (스테이지별) | 적절 |

- 개선 제안: 하트 출현율을 난이도별로 차등 (아기 15%, 전설 3%) (P1)

#### B. UI 구성 분석

**1) StageHUD (상단 정보)**
- 반투명 검정 배경(80px) + 월드명/점수/난이도별/프로그레스바
- 판정: 정보량 적절. 6살이 "숫자가 올라간다"는 인지 가능.
- 문제: 별 카운터(starCount)가 우하단에 작게 표시 - 6살이 인식 어려움
- 개선: 별 아이콘을 크게 + 수집 시 팝업 연출 강화 (P3)

**2) HeartHUD (하트 표시)**
- 왼쪽 상단, 크기 28px, 간격 56px
- Graphics API로 하트 모양 직접 그림 (원+삼각형)
- 판정: 크기/위치 적절. 빨강->회색 전환 직관적.
- 잠재 문제: 하트가 상단 HUD 배경 안에 있어서 월드 이름과 겹칠 수 있음
- y=72 위치인데, HUD 배경이 80px 높이 -> 간신히 들어감

**3) PowerUpHUD (파워업 표시)**
- 공룡 머리 위에 아이콘+라벨+시간바
- 50ms마다 위치 갱신 (공룡 추적)
- 판정: 직관적. 공룡 위에 뜨니까 눈에 잘 띔.
- 문제 없음.

**4) JumpGuideHUD (점프 안내)**
- "화면을 터치하면 점프!" 텍스트, 5초 후 사라짐
- 판정: 단순하고 명확. 6살 적합.
- 개선: 첫 플레이 시만 표시, 이후 숨기기 (P3)

**5) SelectScene (캐릭터 선택)**
- 2x2 그리드 카드 + 미리보기 영역 + 시작 버튼
- 판정: 레이아웃 적절. 카드 크기(110px)가 터치하기 충분.
- 미리보기가 능력 설명을 포함해서 좋음.
- 잠재 문제: 공룡 선택 안 하면 버튼 회색인데 "왜 안 눌리지?" 혼란 가능
- 개선: 기본 선택(브라키오) 자동 적용 또는 "먼저 골라줘!" 깜빡임 (P2)

**6) DifficultyScene (난이도 선택)**
- 5개 세로 카드, 성장 단계 시각화 (카드 크기 점진 증가)
- 판정: 직관적. 별 개수+이모지로 난이도 구분 명확.
- 문제: 5개가 세로로 빽빽함. 작은 화면에서 카드가 겹칠 수 있음.
- 개선: 아기/꼬마/씩씩한 3개만 기본 표시, 용감/전설은 조건부 해금 (P3)

**7) WorldMapScene (월드맵)**
- 좌우 스와이프 + 포커스 카드 + 스테이지 버튼 5개
- 판정: 구조 좋음. 잠긴 월드 시각적으로 명확.
- 문제: 월드 내 스테이지 버튼(원형)이 작음 (반지름 24px)
- 개선: 스테이지 버튼 크기 키우기 + 현재 스테이지 강조 (P2)

**8) PauseScene (일시정지)**
- 반투명 오버레이 + 3버튼 (계속/다시/나가기)
- ESC/P 키 지원
- 판정: 깔끔. 기능 충분.
- 문제: 일시정지 진입 버튼(||)이 HUD 반투명 배경 안에 있어 눈에 안 띌 수 있음
- 개선: 일시정지 버튼 색상 강화 (P3)

**9) GameOverScene (게임오버)**
- "앗!" + 넘어진 공룡 + 점수 + 재도전/월드맵/공룡바꾸기 버튼
- 판정: 연출 훌륭. 바운스/흔들림 효과가 6살에게 재미.
- 문제: 버튼 3개가 세로로 빽빽 (70%, 80%, 90% 위치)
- 개선: 재도전 버튼을 더 크게 + 다른 버튼 작게 (P3)

**10) StageClearScene (클리어)**
- 별 1~3개 연출 + 기뻐하는 공룡 + 다음 스테이지 버튼
- 판정: 축하 연출 훌륭. 별 회전 등장 효과 좋음.
- 문제 없음.
