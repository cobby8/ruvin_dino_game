#### [B] 밸런스 분석 (물리 수치 검증)

점프 높이 계산 (h = v^2 / 2g, 중력 g=800):
| 점프 종류 | 속도(v) | 최대 높이 | 용도 |
|----------|---------|----------|------|
| 낮은 점프 | -450 | 126px | 작은 장애물 (최대 68px) -> 여유 58px OK |
| 높은 점프 | -700 | 306px | 큰 장애물 + 높은 아이템 -> 충분 |
| 2단 점프 | -500 | 156px 추가 | 긴급 회피 -> 합산 460px+ 충분 |
| 스프링 | -900 | 506px | 최고 높이 아이템용 -> 충분 |

판정: 낮은 점프로 모든 일반 장애물(최대 68px)을 넉넉히(+58px) 넘을 수 있음.
높은 점프는 약 4.5배 여유로 과도하게 높음 -> 높은 곳 아이템 보상으로 활용 중이라 적정.

속도 밸런스:
| 난이도 | 시작속도 | 최대속도 | 장애물 간격(ms) | 반응시간(60fps) |
|--------|---------|---------|----------------|----------------|
| 아기(1) | 130 | 260 | 2500~4000 | ~2.6초 (매우 여유) |
| 꼬마(2) | 156 | 325 | 2000~3500 | ~1.8초 (여유) |
| 씩씩(3) | 182 | 390 | 1800~3000 | ~1.3초 (적정) |
| 용감(4) | 208 | 455 | 1500~2500 | ~1.0초 (빠듯) |
| 전설(5) | 234 | 520 | 1200~2000 | ~0.7초 (도전적) |

판정: 난이도 간 차이가 뚜렷하고 6살 기준 아기/꼬마가 충분히 여유로움. 밸런스 양호.

문제점 3가지:
1. 스테이지 후반 속도 증가가 느림 (SPEED_INCREMENT=2, 50점마다 10 증가 -> 체감 미미)
2. 장애물 간격 감소가 clearScore 기준이라 후반 급격히 빨라질 수 있음 (최소 800ms)
3. 적 스폰 빈도가 스테이지별로만 제어되고 시간 경과에 따른 긴장감 상승이 없음

#### [F] 주요 설계 상세

**1. 콤보 시스템**
비유: 볼링에서 스트라이크를 연속으로 치면 점수가 더 올라가는 것처럼.
- 연속 장애물 넘기/적 처치 시 배율 증가: x1 -> x2 -> x3 -> x5 (최대)
- 2초 이내에 다음 성공을 해야 콤보 유지 (타이머 초기화)
- 화면에 "x2 콤보!" "x3 콤보!" 텍스트 팝업
- 콤보 중 별 획득량도 배율 적용
- config.js에 COMBO 상수 추가: { TIMEOUT: 2000, MAX_MULTI: 5, MULTIPLIERS: [1,2,3,5] }

**2. 공룡별 특수능력**
비유: 게임 캐릭터마다 고유 스킬이 있는 것. 마리오 vs 루이지처럼.
- 브라키오(초록): 높은 점프 (HIGH_VELOCITY x1.2) - 목이 긴 초식공룡이라 높이 뜀
- 티라노(주황): 강한 공격 (적 처치 시 별 2배) - 가장 강한 공룡이라 공격력 높음
- 트리케라(보라): 방어력 (피격 무적 시간 x1.5) - 방패 뿔이 있어서 오래 버팀
- 프테라노(파랑): 활강 (하강 속도 x0.7, 천천히 내려옴) - 날개가 있어서 느리게 착지
- 구현: Dino.js에 dinoKey별 분기, config.js에 DINO_ABILITIES 상수

**3. 밸런스 최적화 제안값**
| 항목 | 현재값 | 제안값 | 이유 |
|------|--------|--------|------|
| SPEED_INCREMENT | 2 (10점마다) | 5 (10점마다) | 후반 속도감 부족, 2.5배 증가 |
| 장애물 간격 최소 | 800ms | 1000ms | 전설 난이도에서도 반응 가능하도록 |
| 적 스폰 딜레이 | 고정 3000ms | 3000ms -> 시간 경과에 따라 2000ms까지 감소 | 긴장감 점진적 상승 |
| 스테이지 후반 maxSpeed | 난이도별 고정 | 스테이지 speedBonus만큼 추가 상한 | 후반 월드가 더 빠르게 느껴지도록 |

#### [G] developer 주의사항

1. **파티클은 Phaser 내장 ParticleEmitter 사용**: 외부 라이브러리 금지. this.add.particles() 사용.
2. **카메라 페이드는 this.cameras.main.fade/fadeIn 사용**: 씬 전환 시 300ms fadeOut -> scene.start -> 300ms fadeIn 패턴.
3. **콤보 타이머는 time.delayedCall이 아닌 수동 관리**: update()에서 this.time.now 체크. delayedCall은 pause 시 멈추지 않음.
4. **공룡 특수능력은 config.js에 상수화**: 하드코딩 금지. DINO_ABILITIES = { brachio: { jumpMulti: 1.2 }, ... } 형태.
5. **튜토리얼은 오버레이 방식**: 별도 씬이 아닌 GameScene 위에 투명 오버레이로 안내. 첫 플레이(localStorage 체크) 시에만 표시.
6. **BGM은 월드별 다른 키/템포로 생성**: SoundGenerator에 playWorldBGM(worldId) 메서드 추가. 기존 playBGM 구조를 확장.
