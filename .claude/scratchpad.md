## 기획설계 (planner-architect)

### 분석 1: 적 밟기 판정이 좁은 문제 (2026-03-28)

**현재 밟기 판정 조건** (GameScene.js 708행):
- `velocity.y > 0` (아래로 떨어지는 중) AND `dino.body.bottom <= enemy.body.top + 30`
- 허용 범위: 30px

**적 히트박스** (Enemy.js, 이미지 사용 시):
- 원본 프레임: 160x720 (세로 여백 매우 큼)
- 히트박스: 48x48px (원본), 스케일 0.75 적용 -> 화면상 36x36px
- Y오프셋: (720-48)/2 = 336px -> 히트박스가 이미지 중앙에 위치
- **문제: body.top이 이미지 머리가 아니라 중간 높이에 있어 밟기 조건 불일치**

**공룡 히트박스** (Dino.js, 이미지 사용 시):
- 원본 프레임: 516x512, 히트박스 206x256px, 스케일 0.28 -> 화면상 58x72px

**원인 요약:**
1. 적 이미지 720px 중 히트박스가 Y=336에서 시작 -> body.top이 실제 머리보다 훨씬 아래
2. 히트박스 36x36px로 너무 작아 충돌 자체가 안 일어남
3. 720px 높이에 대해 160px 기준으로 히트박스를 계산하여 비율 불일치

**해결 방향:**
- 히트박스 Y오프셋을 이미지 상단 쪽으로 올려야 함 (적의 실제 머리 위치에 맞춤)
- 히트박스 크기를 좀 더 키우거나, 밟기 허용 범위(30px)를 더 늘려야 함
- bodyH 계산에 720 기준을 사용하여 세로 히트박스를 넓혀야 함
