#### 2. 물리 계산 (정확한 수치)

**공식: 최대높이 = V^2 / (2 x G), 체공시간 = 2 x |V| / G**

| 점프 종류 | 초속도 | 최대 높이 | 체공 시간 | 비유 |
|----------|--------|----------|----------|------|
| 낮은 점프 | -450 | **126.6px** | **1.125초** | 살짝 깡충 |
| 높은 점프 | -700 | **306.3px** | **1.75초** | 크게 뛰기 |
| 2단 점프 (추가분) | -500 | **+156.3px** | 추가 1.25초 | 공중에서 한번 더 |
| 브라키오 낮은 점프 | -540 (x1.2) | **182.3px** | **1.35초** | 특수능력: 20% 높게 |
| 브라키오 높은 점프 | -840 (x1.2) | **441.0px** | **2.10초** | 특수능력: 20% 높게 |
| 스프링 점프 | -900 | **506.3px** | **2.25초** | 최고 높이! |

**홀드 부스트 타이밍 검증:**
- 150ms 시점 속도 = -450 + 800 x 0.15 = **-330** (아직 음수 = 상승 중)
- 부스트 조건 충족됨 (velocity.y < 0)
- 150ms 시점에서 속도를 -700으로 교체 --> 높은 점프 발동 OK

**1단 낮은점프 + 2단 점프 총 도달 높이 (최악의 경우: 정점 근처에서 2단):**
- 1단 정점(126.6px)에서 속도=0인 상태로 2단 점프(-500) 발동
- 추가 상승 = 500^2 / (2x800) = 156.3px
- **총 = 126.6 + 156.3 = 282.9px**

**1단 높은점프 + 2단 점프 총 도달 높이:**
- 1단 정점(306.3px)에서 2단 발동
- **총 = 306.3 + 156.3 = 462.6px**

- 장애물 스폰 = 화면 오른쪽 밖 (width + 50 정도)
- 이동 거리 = width x 0.8 + 50 = 약 350px (375px 폭 기준)

- 씩씩한공룡 기준: 100점 도달 시 182 + 50 = 232 --> 도달 시간 1.51초
- 전설의공룡 기준: 100점 도달 시 234 + 50 = 284 --> 도달 시간 1.23초

**6살 기준 반응 시간 분석:**
- 6세 아동 평균 반응 시간: ~500ms (성인 ~250ms의 약 2배)
- 반응 + 점프 실행까지: ~600-700ms
- 아기공룡(2.69초): 매우 충분 (여유 2초)
- 전설의공룡 후반(1.23초): 6살에게는 빡빡하지만, "전설" 난이도니까 의도된 것

**결론: 반응 시간은 모든 난이도에서 6살도 가능한 수준**
- 아기~씩씩한: 매우 여유로움
- 용감한~전설: 어렵지만 불가능하지는 않음

#### 5. 높은 점프 체감 분석

**현재 메커니즘:**
1. 누르는 순간(DOWN) --> 즉시 LOW_VELOCITY(-450)로 점프
2. 떼는 순간(UP) --> 150ms 이상 눌렀으면 HIGH_VELOCITY(-700)로 교체

**문제 발견 2: "길게 누르기"가 6살에게 직관적이지 않을 수 있음**
- 150ms는 매우 짧음 (사람이 의식적으로 구분하기 어려운 시간)
- 6살: "꾹 누르면 높이 뛴다"를 이해해도, 실제로는 대부분 300-500ms 누름
- 결과적으로 거의 모든 점프가 높은 점프가 될 가능성 높음
- 현재 150ms HOLD_THRESHOLD는 사실상 "빠르게 탭 vs 보통 탭"을 구분하는 것

**실제 체감:**
- 스마트폰에서 빠른 탭 = ~80-120ms --> 낮은 점프 발동
- 보통 탭/의식적 누르기 = ~200-500ms --> 높은 점프 발동
- 이 구분 자체는 **괜찮은 편** (빠르게 톡 치면 낮게, 꾹 누르면 높게)

**시각 피드백 부재:**
- 현재 낮은/높은 점프의 시각적 차이가 **없음** (같은 jump 애니메이션)
- 2단 점프도 시각적 피드백 없음
- 플레이어가 "내가 높은 점프를 했는지" 알기 어려움

---

#### 6. 2단 점프 체감 분석

**발동 조건:**
- 공중에 있을 때 + 아직 2단 점프를 안 썼을 때
- 난이도별: 아기~씩씩한=무제한, 용감한=3회/스테이지, 전설=불가

**실제 도달 높이:**
- 1단 낮은 + 2단 = 최대 283px (높은 점프 306px보다 낮음)
- 1단 높은 + 2단 = 최대 463px (스프링 506px 다음으로 높음)
- 주요 용도: 비행 적 처리, 긴급 회피

**문제 발견 3: 2단 점프가 일반 높은 점프와 체감 차이가 작음**
- 낮은점프+2단(283px) vs 높은점프(306px) --> 거의 같은 높이
- 아이가 "2단 점프를 해야 하는 상황"을 인식하기 어려움
- 2단 점프는 주로 "높은 점프를 못 했을 때의 보험"으로 작동

#### 8. 개선안 제안

##### A. 즉시 수정 (obstacleScale 미적용 버그)

**변경 전:** difficulties.js에 obstacleScale이 정의되어 있지만, Obstacle.js/GameScene.js 어디에서도 사용하지 않음
**변경 후:** 장애물 스폰 시 난이도의 obstacleScale을 곱해서 적용

| 난이도 | obstacleScale | 효과 |
|--------|-------------|------|
| 아기공룡 | 0.8 | 장애물 20% 작음 --> 더 쉽게 넘기 |
| 꼬마공룡 | 0.9 | 장애물 10% 작음 |
| 씩씩한공룡 | 1.0 | 기본 크기 |
| 용감한공룡 | 1.1 | 장애물 10% 큼 --> 더 어려움 |
| 전설의공룡 | 1.2 | 장애물 20% 큼 --> 가장 어려움 |

구현: GameScene에서 ObstacleManager에 obstacleScale을 전달, spawn 시 setScale에 반영

##### B. 비행 적 높이 조정

**변경 전:** 독수리 flyHeight = 0.4 (바닥에서 280px)
**변경 후:** 독수리 flyHeight = 0.45 (바닥에서 247px)

이유: 높은 점프(306px)로 여유있게 닿을 수 있도록 (여유 26px --> 59px)

##### C. 2단 점프 강화

**변경 전:** DOUBLE_VELOCITY = -500 (추가 156px)
**변경 후:** DOUBLE_VELOCITY = -550 (추가 189px)

효과:
- 낮은점프+2단 = 126 + 189 = **315px** (높은 점프 306px보다 약간 높음 --> 2단 점프의 가치 생김)
- 높은점프+2단 = 306 + 189 = **495px** (스프링 506px에 근접 --> 최고 높이 느낌)

##### D. 시각적 피드백 추가 (선택사항)

1. **높은 점프 시**: 점프 순간 작은 바람 이펙트 (기존 파티클 시스템 활용)
2. **2단 점프 시**: 공중에서 작은 원형 이펙트 (flash)
3. **착지 시**: 먼지 이펙트 (높은 점프/2단 점프 후에만)

##### E. 난이도별 점프 차별화 (미채택 -- 현재로는 불필요)

이유: 현재 점프 높이는 모든 난이도에서 균일하되, obstacleScale로 장애물 크기를 조절하는 방식이 더 직관적
- 점프 높이까지 바꾸면 공룡의 "감각"이 난이도마다 달라져 혼란스러움
- obstacleScale만 적용하면 "같은 공룡이 더 큰/작은 장애물을 만남"으로 이해 가능

##### F. 홀드 부스트 개선 (미채택 -- 현재로는 충분)

이유: 현재 150ms 구분 기준은 실제 플레이에서 잘 작동함
- 빠른 탭(80-120ms) = 낮은 점프, 보통 탭(200ms+) = 높은 점프
- 단계적 점프(150/300/500ms)는 6살에게 오히려 복잡함
